# 🚀 100 Days of DevOps Challenge

Welcome to my **100 Days of DevOps Challenge** repository!

This repository documents my journey of consistently learning and implementing DevOps concepts through hands-on tasks over 100 days.

---

## 🎯 Objective

* Build strong DevOps fundamentals from scratch
* Gain hands-on experience with real-world scenarios
* Master tools like Linux, Git, Docker, Kubernetes, Jenkins, Ansible, and Terraform
* Improve troubleshooting and automation skills

---

## 📂 Repository Structure

Each day contains:

* 📌 Task description
* ⚙️ Step-by-step implementation
* 💻 Commands and configurations
* 🧠 Key learnings

---

## 📅 Challenge Breakdown

### 🐧 Linux (Day 1–15)

* User management & security
* SSH configuration
* Cron jobs & scripting
* SELinux, networking, process troubleshooting

### 🌐 Web & Database Servers (Day 16–20)

* Nginx, Tomcat, PostgreSQL
* Load balancing & SSL setup
* Application deployment

### 🔧 Git (Day 21–34)

* Repository management
* Branching & merging
* Rebase, stash, cherry-pick
* Conflict resolution & hooks

### 🐳 Docker (Day 35–47)

* Containerization basics
* Dockerfiles & images
* Networking & Compose
* Application deployment in containers

### ☸️ Kubernetes (Day 48–67)

* Pods, Deployments, Services
* Rolling updates & rollback
* Volumes, secrets, sidecars
* Real-world app deployments

### ⚙️ Jenkins (Day 68–81)

* CI/CD pipelines
* Job automation
* Plugins & security
* Multi-stage pipelines

### 🤖 Ansible (Day 82–93)

* Inventory & playbooks
* Modules (ping, file, service)
* Automation & configuration management
* Jinja2 templates & conditionals

### ☁️ Terraform (Day 94–100)

* Infrastructure as Code (IaC)
* VPC, EC2, IAM
* Security groups
* CloudWatch monitoring

---
